linkpulse api canary rollout
============================

the api workload runs as an argo rollouts Rollout (chart argo-rollouts 2.41.0,
controller v1.9.0). the canary strategy is replica ratio based with no traffic
routing: the ingress reaches the api through the web nginx and the api Service,
so a weight is expressed as a share of the api pods.

strategy steps:
  setWeight 20  ->  pause 60s  ->  setWeight 50  ->  pause 60s  ->  full

the progression below was captured while the gitops loop delivered a new image
tag (bot pr bumping the tag in gitops/apps/dev/linkpulse/values.yaml, merged,
argocd synced). the old stable replicaset is 66957677bd, the new canary
replicaset is 559cfd6d95. the hpa floor was 2 replicas during this run, so 20
percent rounds to one canary pod against two stable, and 50 percent is one
canary against one stable.
