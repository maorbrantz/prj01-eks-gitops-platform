grafana dashboards and data flow
================================

grafana is reached over a port-forward (no ingress this phase):

  kubectl -n monitoring port-forward svc/kube-prometheus-stack-grafana 3000:80

the admin password is generated by the chart into a secret and is not in git.
retrieve it with:

  kubectl -n monitoring get secret kube-prometheus-stack-grafana \
    -o jsonpath='{.data.admin-password}' | base64 -d

the two project dashboards ship as code (ConfigMaps labeled grafana_dashboard=1,
imported by the grafana sidecar) and show up alongside the stock stack boards:

  LinkPulse Golden Signals (uid linkpulse-golden)
  Platform Overview        (uid platform-overview)

all three datasources are wired:

  Alertmanager (alertmanager)
  Loki         (loki)
  Prometheus   (prometheus)

data flows end to end. querying the golden-signals panels through grafana's own
prometheus datasource proxy returns live values from the api /metrics scrape:

  api requests per second = 7.3088
  api p95 latency seconds  = 2.2502   (elevated here because a load test had just run)

logs flow too. promtail ships pod logs to loki, and the loki datasource returns
streams for the app namespace:

  loki has log streams for linkpulse-dev, sample line:
    INFO:     10.0.56.197:46026 - "GET /readyz HTTP/1.1" 200 OK
